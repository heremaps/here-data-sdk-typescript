Class MultiPartUploadWrapper

Uploads data to the blob storage in chunks.

You can use this class in browsers and Node.js to upload large files.

A file is not loaded in RAM but is read and uploaded in chunks. It enables uploading large files.

You can create an instance of this class in browser or Node.js.

If you want to read data from a different place, implement and set the BlobData class.

    • Defined in @here/olp-sdk-dataservice-write/lib/utils/MultiPartUploadWrapper.ts:220

    Uploads data in chunks to the blob storage.

    Starts the MultiPartUpload process, uploads the data in chunks, and then completes the MultiPart Upload process. The data is uploaded to Blob API v1 or v2.

    resurns

    The Promise with the HTTP status or rejects in case of errors.

    Parameters

    • data: string | ArrayBuffer | Blob | BlobData | SharedArrayBuffer | File

      The data to upload.

      Use the following values:

      • In browsers: File | Blob | ArrayBufferLike.
      • In Node.js: string (the path of the file) | ArrayBufferLike.

      Also, you can provide your own implementation of the BlobData class, which enables you to use various sources of data.

    • Optional abotrSignal: AbortSignal

    Returns Promise<number>
